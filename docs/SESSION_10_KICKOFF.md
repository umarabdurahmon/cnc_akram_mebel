## Промпт

Прочитай `CLAUDE.md` и `docs/тз.md` (разделы 7.4 пп. 3–4, 9.11). Сначала план,
потом код. Ревью перед миграцией. Эндпоинты — только роль `manager`.

### 1. `operating_expense` — операционные расходы (тз 9.11)

Расход вне заказа (аренда, коммуналка, связь):
`category_id` FK → `expense_category`, `amount` NUMERIC(12,2), `spent_on` DATE,
`note` (null), `created_by` (FK). Тот же справочник категорий. В v1 вносится
вручную. CRUD, manager. Миграция — на ревью.

### 2. Месячный отчёт на лету (тз 7.4.4 — главный камень)

- доход = `SUM(order_payment.amount)` за месяц;
- расход = `SUM(order_expense)` + `SUM(operating_expense)` за месяц, с разбивкой
  по категориям;
- прибыль = доход − (расходы по заказам + операционные);
- НЕ хранить — считать запросом по требованию.

### 3. Сборка из трёх источников — не задвоить и не потерять

- разбивка по категориям объединяет ОБА расходных источника (`order_expense` И
  `operating_expense`), сгруппированные по категории; категорию, присутствующую
  только в одном источнике, НЕ терять (объединение/агрегация, не inner join);
- включать деактивированные категории, если у них есть исторические расходы за
  период (мягкое удаление их сохранило); фильтрация по `is_active` в отчёте их
  потеряла бы;
- внутренний инвариант: итог расходов == сумма разбивки по категориям ==
  `SUM(order_expense) + SUM(operating_expense)`.

### 4. Точность и границы месяца

- суммирование в SQL (`SUM` по `NUMERIC`, точно); в API всё остаётся `Decimal`;
- даты тут `DATE` (`spent_on`/`received_on`) → конвертация часового пояса НЕ
  нужна (в отличие от посещений); диапазон месяца — чистый `BETWEEN`;
- месяц без данных → нули, не ошибка/null.

### 5. Несущие тесты (только эти, настоящий Postgres, через auth-фикстуру)

- доход / расход / прибыль за месяц считаются верно из трёх источников;
- инвариант: итог расходов == сумма разбивки по категориям;
- категория только с операционными (или только с заказными) расходами НЕ теряется
  в разбивке;
- деактивированная категория с историческим расходом за месяц присутствует в
  разбивке;
- месяц без данных → нули;
- суммы `Decimal`, без потери точности;
- операционные расходы и отчёт — только `manager`.

### Definition of Done (Сессия 10)

- Руководитель вносит операционные расходы; месячный отчёт считается на лету
  (доход, расход с разбивкой по категориям, прибыль) из трёх источников без
  задвоений и потерь; деактивированные категории с историей видны в разбивке.
- `make migrate` чисто; `make check` зелёный, показан вывод.
- Закоммичено мелкими логическими коммитами.
