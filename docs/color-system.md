# Цветовая система фронтенда

Все токены определены в `frontend/src/App.vue` в глобальном блоке `<style>` (`:root`).
Ни один `.vue`-файл не использует жёстко заданные цвета — только переменные ниже.

---

## Telegram theme wrappers

Короткие псевдонимы для динамических переменных Telegram WebApp.
Fallback-значения — на случай запуска вне Telegram (разработка в браузере).

| Токен           | Источник                                  | Fallback  | Назначение                        |
|-----------------|-------------------------------------------|-----------|-----------------------------------|
| `--c-bg`        | `--tg-theme-bg-color`                     | `#ffffff` | Основной фон страницы             |
| `--c-surface`   | `--tg-theme-secondary-bg-color`           | `#f2f2f7` | Карточки, поля ввода, секции      |
| `--c-text`      | `--tg-theme-text-color`                   | `#000000` | Основной текст                    |
| `--c-hint`      | `--tg-theme-hint-color`                   | `#8e8e93` | Подсказки, второстепенный текст   |
| `--c-accent`    | `--tg-theme-button-color`                 | `#2481cc` | Кнопки-действия, активные табы    |
| `--c-accent-text` | `--tg-theme-button-text-color`          | `#ffffff` | Текст на кнопках-акцентах         |

> В тёмной теме Telegram автоматически переопределяет `--tg-theme-*` — все компоненты
> адаптируются без дополнительного кода.

## Semantic colors

Фиксированные цвета, смысл которых не зависит от темы.

### Positive (доход / подтверждено / активно)

| Токен                | Значение    | Где используется                                        |
|----------------------|-------------|---------------------------------------------------------|
| `--c-positive`       | `#27ae60`   | Суммы оплат, статус «подтверждено», активный сотрудник |
| `--c-positive-bg`    | `#eafaf1`   | Фон кнопки «Подтвердить» (неактивное состояние)        |
| `--c-positive-dim`   | `#d5f5e3`   | Фон бейджа «confirmed» в списке посещений              |
| `--c-positive-dark`  | `#1e8449`   | Текст на светло-зелёном фоне (бейджи)                  |

### Negative (расход / ошибка / расхождение)

| Токен                | Значение    | Где используется                                         |
|----------------------|-------------|----------------------------------------------------------|
| `--c-negative`       | `#e74c3c`   | Суммы расходов, ошибки, кнопка удаления                 |
| `--c-negative-bg`    | `#fde8e8`   | Светлый фон для ошибок и дедлайна-просрочки             |
| `--c-negative-dim`   | `#fdecea`   | Фон кнопки «Расхождение» / аватара в списке расхождений |
| `--c-negative-dark`  | `#c0392b`   | Текст на светло-красном фоне                            |

### Warning (предупреждение / операционные расходы)

| Токен                | Значение    | Где используется                              |
|----------------------|-------------|-----------------------------------------------|
| `--c-warning`        | `#e67e22`   | Предупреждения в формах, бар операционных расходов |
| `--c-warning-bg`     | `#fff3e0`   | Фон пилюли «urgent» дедлайна                 |

## WorkerView "on duty" gradient

Специальные цвета для состояния «сотрудник на смене» в `WorkerView`.
Намеренно яркие — визуальный акцент на активном состоянии.

| Токен                | Значение    | Назначение                              |
|----------------------|-------------|-----------------------------------------|
| `--c-checkin-from`   | `#1b5e20`   | Начало градиента карточки               |
| `--c-checkin-to`     | `#2e7d32`   | Конец градиента карточки                |
| `--c-checkin-pulse`  | `#69f0ae`   | Пульсирующая точка активного статуса   |

> Pulse-анимация использует `rgba(105,240,174, ...)` — производные от `--c-checkin-pulse`.
> CSS `color-mix()` здесь не применяется ради совместимости с WebView.

---

## Исключения (намеренно жёстко заданные цвета)

- **Stage pill** в `OrdersView` (`#e3f2fd / #1565c0`) — цвет этапа производства.
  Это informational-цвет (не semantic), и он один на весь UI — вынесение в токен
  не даёт выгоды.
- **rgba-значения в pulse-анимации** — CSS-переменные не поддерживаются как часть
  `rgba()` в старых WebView; значения закомментированы со ссылкой на `--c-positive`.

---

## Правила при добавлении нового компонента

1. Никаких hex/rgb/rgba напрямую в `<style scoped>` — только токены выше.
2. Новый семантический цвет → добавить токен в `:root` в `App.vue` + строку в этот файл.
3. Тема (светлая/тёмная) → используй `--c-bg` / `--c-surface` / `--c-text` / `--c-hint`.
   Не добавляй `[data-theme="dark"]` overrides — Telegram передаёт правильные значения автоматически.
4. Акцентный цвет — всегда `--c-accent` / `--c-accent-text`. Не подставляй `#2481cc` напрямую.
