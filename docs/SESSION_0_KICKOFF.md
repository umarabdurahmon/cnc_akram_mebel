## Промпт

Прочитай `CLAUDE.md` и `docs/тз.md` — это контекст и требования проекта.

Твоя задача в этой сессии — собрать **шагающий скелет**: пустой каркас без
бизнес-логики, который работает насквозь через весь стек. Никаких моделей
посещений/заказов/финансов пока — только обвязка.

Сделай по порядку, останавливаясь на ревью перед каждым крупным шагом:

1. **Бэкенд `server/`**: инициализируй Python-проект (`pyproject.toml`) с
   FastAPI, SQLAlchemy 2.x, Alembic, aiogram 3.x, pytest, ruff, mypy, psycopg.
   Настрой ruff и mypy. Структура пакета: `app/{core,db,models,schemas,
   repositories,services,api}`, `app/main.py` (FastAPI), `bot/main.py` (aiogram),
   `alembic/`, `tests/`. Поставь актуальные стабильные версии.

2. **Health-эндпоинт**: `GET /api/health` возвращает `{"status": "ok"}` и
   проверяет соединение с БД. Подключение к Postgres — из `DATABASE_URL`.

3. **Alembic**: инициализируй, настрой на `DATABASE_URL`. Пустая первая
   ревизия (без таблиц пока) — просто чтобы убедиться, что миграции бегут.

4. **Бот-заглушка**: `bot/main.py` на long-polling, отвечает на `/start`. Токен
   из `BOT_TOKEN`. Бизнес-логики нет.

5. **Фронтенд `frontend/`**: инициализируй Vue 3 + Vite (vite.config.js уже
   есть, не перезаписывай прокси и allowedHosts). Простая страница, которая
   дёргает `/api/health` и показывает результат. Настрой eslint.

6. **Петля верификации**: добей `make check`, чтобы он проходил зелёным на
   пустом каркасе (ruff + mypy + pytest + сборка фронта). Один dummy-тест на
   health-эндпоинт против тестовой БД (`TEST_DATABASE_URL`), в транзакции с
   откатом — чтобы проверить, что тестовая обвязка работает.

### Definition of Done (Сессия 0)

- `docker compose up -d postgres` поднимает БД, health-эндпоинт видит её.
- `make check` зелёный, показан его реальный вывод.
- Vite-страница открывается через тоннель (ngrok/cloudflared) внутри Telegram
  как Mini App и успешно зовёт `/api/health` через прокси.
- Бот отвечает на `/start`.
- Всё закоммичено мелкими логическими коммитами в `main`.

Не переходи к моделям и реальным фичам в этой сессии — это Сессия 1 (авторизация
+ `employee` + первая содержательная миграция).
