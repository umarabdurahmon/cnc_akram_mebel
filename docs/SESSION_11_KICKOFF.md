# Сессия 11 — Финансы: Vue-дашборд + ввод в карточке (финальная)

> Предусловие: Сессии 0–10 пройдены. Весь бэкенд готов (посещения, заказы,
> финансы). Это финальная сессия: интерфейс финансов + выход в прод. Всё —
> manager-only.

## Промпт

Прочитай `CLAUDE.md` и `docs/тз.md` (раздел 7.4). Сначала план, потом код. Строй
на фундаменте Сессий 3/8 (WebApp, api-клиент, i18n). Подписи переводим; контент
и имена категорий — нет.

### 1. Ввод денег без float (камень)

- суммы в форме держать СТРОКОЙ от ввода до отправки; НЕ `parseFloat → number →
  API` (float теряет точность на больших UZS-суммах);
- отправлять строкой — под контракт «`Decimal` как строка» из Сессии 9;
- UZS целочисленна — ввод можно без дробной части, контракт всё равно строковый;
- учесть локаль ввода (разделитель), валидировать сумму;
- для ОТОБРАЖЕНИЯ `Intl.NumberFormat` безопасен (разделители разрядов).

### 2. Дашборд месячного отчёта

- доход, расход с разбивкой по категориям, прибыль (может быть отрицательной —
  показать знаком/цветом);
- переключатель месяца;
- тянуть отчёт свежим каждый раз (на лету, без клиентского кеша) — после ввода
  цифры обновляются;
- i18n подписей.

### 3. Ввод в карточке заказа (manager, тз 7.4)

- оплаты: сумма, дата, заметка;
- двухуровневый расход: направление (`to_employee`/`general`); при `to_employee`
  форма требует сотрудника (зеркалит CHECK для UX, но авторитет — бэкенд: 400
  обрабатывать мягко); категория из активных.

### 4. Операционные расходы (форма, manager)

UI ввода операционных расходов из Сессии 10.

### 5. Проверка

- `make check` (eslint + сборка) зелёный;
- полная проверка вручную в Telegram через тоннель.

### 6. Go-live (после сессии)

- прод: домен + SSL (Let's Encrypt) фронтит Mini App и API (nginx);
- Postgres + два процесса (API, бот) на VPS — systemd или Docker Compose;
- бот: long-polling достаточно для одного предприятия (или webhook);
- прод-`.env` с реальными секретами; URL Mini App (прод-домен) в BotFather;
  засидить реального руководителя (`make seed-manager`);
- бэкапы: `pg_dump` по cron + бэкап файлового хранилища (файлы на локальной ФС,
  пока не на S3);
- финальный `make check` по всему проекту.

### Definition of Done (Сессия 11 + проект)

- Руководитель ведёт оплаты и расходы в карточке заказа и операционные расходы;
  видит месячный отчёт в дашборде (доход / расход по категориям / прибыль) с
  переключением месяца; деньги вводятся без потери точности.
- `make check` зелёный, показан вывод; закоммичено.
- Этап 3 ТЗ закрыт. Проект функционально завершён: посещения, заказы, финансы —
  на бэкенде, в боте и в Mini App, двуязычно (ru/uz), за HTTPS.
